The APGI API provides access to a scientifically-validated framework for understanding consciousness and cognitive processing. Built on the Allostatic Precision-Gated Ignition model, this API enables developers to integrate consciousness assessment, cognitive profiling, and real-time state analysis into their applications.

The API supports analysis of psychological assessments, cognitive parameter estimation, profile comparison, and state prediction based on validated APGI framework principles combining predictive processing, thermodynamics, and global workspace theory.

Key Features

  • Scientifically-validated cognitive parameter estimation (ε, π, θₜ, β)
  • Real-time consciousness state analysis across 43+ psychological states
  • Profile generation from questionnaire responses or behavioral data
  • Comparative analysis between profiles with similarity scoring
  • State transition prediction based on parameter changes
  • Support for multiple data formats (JSON, CSV, streaming)

To begin using the APGI API, you'll need to obtain an API key and configure your development environment. The API is RESTful and returns JSON-formatted responses.

Quick Start

Step 1: Get Your API Key

Register for an API key at the APGI Developer Portal. Keys are environment-specific (development, staging, production).

Step 2: Install Client Library (Optional)

We provide official client libraries for popular languages: 

# Python
pip install apgi-api

# JavaScript/Node.js
npm install apgi-api

# R
install.packages("apgi")

Step 3: Make Your First Request

Test your connection with a simple profile analysis:

curl -X POST https://api.apgi.io/v1/analyze \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "responses": [4, 3, 5, 2, 4, ...],
    "return_profile": true
  }'
Base URL: https://api.apgi.io/v1
Rate Limits: 1000 requests/hour (standard tier), 10,000 requests/hour (premium tier)

The APGI API uses API key authentication via Bearer tokens. All requests must include your API key in the Authorization header.

Authentication Methods

Bearer Token (Recommended)

Include your API key in the Authorization header:

Authorization: Bearer apgi_live_abc123def456ghi789

API Key Formats

  • apgi_test_* - Testing environment (limited features, no billing)
  • apgi_dev_* - Development environment (full features, no rate limits)
  • apgi_live_* - Production environment (full features, rate limits apply)
Security Best Practices:
  • Never expose API keys in client-side code or public repositories
  • Rotate keys regularly (recommended: every 90 days)
  • Use environment variables to store keys
  • Implement IP whitelisting for production keys

Example Authentication

// JavaScript
fetch('https://api.apgi.io/v1/analyze', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer apgi_live_abc123',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ responses: [4,3,5,2,4] })
})

POST /v1/analyze

Generate APGI cognitive profile from questionnaire responses.

Request Parameters

Parameter Type Required Description
responses array[integer] Required Array of 35 questionnaire responses (1-5 scale)
return_profile boolean Optional Include full profile analysis (default: true)
include_states boolean Optional Include probable psychological states (default: false)
confidence_threshold float Optional Minimum confidence for profile (0.0-1.0, default: 0.7)

Response 

{
  "profile_id": "prof_abc123",
  "parameters": {
    "epsilon": 0.72,
    "pi": 0.65,
    "theta_t": 0.58,
    "beta": 0.81
  },
  "profile_type": "Sensitive Integrator",
  "confidence_score": 0.89,
  "processing_time_ms": 245,
  "states": [
    {
      "state": "High Interoceptive Awareness",
      "probability": 0.87,
      "description": "Strong attention to bodily signals"
    }
  ]
}

POST /v1/parameters/estimate

Estimate individual APGI parameters from behavioral data.

Request Parameters

Parameter Type Required Description
data_type string Required Type of input data: "questionnaire", "behavioral", "neural"
data object Required Input data formatted according to data_type
parameters array[string] Optional Specific parameters to estimate (default: all four)

POST /v1/profiles/compare

Compare two or more APGI profiles and calculate similarity metrics.

Request Parameters

Parameter Type Required Description
profile_ids array[string] Required Array of 2-10 profile IDs to compare
metrics array[string] Optional Similarity metrics: "euclidean", "cosine", "manhattan"

POST /v1/states/predict

Predict psychological states based on current parameters and context.

Request Parameters

Parameter Type Required Description
profile_id string Required Profile ID for state prediction
context object Optional Environmental or situational context data
top_k integer Optional Number of top states to return (default: 5)

GET /v1/profiles/{profile_id}

Retrieve a previously generated profile by ID.

Path Parameters

Parameter Type Description
profile_id string Unique identifier for the profile

The APGI API uses standard HTTP status codes and provides detailed error messages in JSON format.

HTTP Status Codes

Code Status Description
200 OK Request succeeded
201 Created Resource successfully created
400 Bad Request Invalid request parameters or malformed data
401 Unauthorized Missing or invalid API key
403 Forbidden API key lacks necessary permissions
404 Not Found Requested resource does not exist
422 Unprocessable Entity Request data fails validation
429 Too Many Requests Rate limit exceeded
500 Internal Server Error Unexpected server error
503 Service Unavailable API temporarily unavailable

Error Response Format 

{
  "error": {
    "code": "invalid_parameter",
    "message": "Invalid value for 'responses': expected 35 integers, got 30",
    "param": "responses",
    "type": "validation_error"
  },
  "status": 400
}

Common Error Types

authentication_error

API key is missing, invalid, or expired. Check your Authorization header.

validation_error

Request parameters don't meet schema requirements. Check parameter types and ranges.

rate_limit_error

Too many requests in a short time. Implement exponential backoff or upgrade your plan.

resource_not_found

Requested profile_id or other resource doesn't exist or has been deleted.

Rate Limiting:
When rate limited, the response includes Retry-After header indicating seconds to wait before retrying. Implement exponential backoff for optimal performance.

What is the APGI framework?

APGI (Allostatic Precision-Gated Ignition) is a scientifically-validated framework for understanding consciousness as an emergent property of thermodynamically-constrained neural processing. It combines predictive processing, global workspace theory, and thermodynamic principles to explain how conscious experiences arise from neural computation.

How accurate are the cognitive profiles?

APGI profiles are generated using scientifically-validated assessment methodology with test-retest reliability >0.85. Parameter estimates are validated against established personality inventories and show strong convergent validity with Big Five traits (r > 0.7). Confidence scores indicate the reliability of individual assessments.

Can I use the API for clinical purposes?

The APGI API is designed for research and informational purposes. While based on validated psychological principles, it should not be used as a sole diagnostic tool for clinical decision-making. Always consult qualified mental health professionals for clinical assessment and treatment.

What are the four core parameters?

ε (Epsilon): Prediction Error Sensitivity - responsiveness to unexpected events
π (Pi): Precision Allocation - how you prioritize information sources
θₜ (Theta-t): Ignition Threshold - level needed for conscious awareness
β (Beta): Somatic Bias - attunement to bodily sensations

How is data privacy handled?

All API requests are encrypted via TLS 1.3. Profile data is anonymized and can be deleted on request. We do not share individual-level data with third parties. Aggregate, anonymized data may be used for research purposes. See our Privacy Policy for full details.

What are the rate limits?

Standard tier: 1,000 requests/hour
Premium tier: 10,000 requests/hour
Enterprise tier: Custom limits available

Rate limits reset hourly. Implement caching and batch processing for optimal usage.

Can I integrate APGI with existing psychological assessments?

Yes! The APGI framework is designed to complement existing psychological assessment tools. We provide mapping functions for popular inventories (Big Five, HEXACO, etc.) and can work with behavioral data from various sources. Contact our integration team for custom solutions.
